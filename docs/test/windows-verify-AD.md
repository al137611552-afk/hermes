## 一、块 D Auto-Retry（**重点：唯一行为变更，已默认在 production 生效**）

机制：单个工具调用因**瞬时 IO**（超时/网络抖动/端口占用）失败 → 自动退避重试（默认最多 2 次，等 0.5s、1s）；非瞬时失败（逻辑/语法/鉴权）**不重试**；撞上限把失败回灌模型。

### TC-D1 瞬时失败自动重试并恢复（正向）
- 前置：`config.yaml` 中 `auto_retry: true`（默认）。
- 步骤：普通对话里让 Agent 跑一条**会瞬时失败**的命令。推荐做法二选一：
  - **断网法**：先断开网络，发「用 run_powershell 执行 `curl https://example.com`」，在第 1 次失败后、约 0.5–1.5s 内**恢复网络**。
  - **占端口法**：先手动占住某端口（如 `python -m http.server 8000` 占 8000），让 Agent 跑一条会因端口占用瞬时失败、随后你释放端口的命令。
- 预期：
  - 对话流出现**重试提示**（`tool_retry` 事件，标 attempt/delay/reason=transient_io），**不**向你提问、**不**把第一次失败当最终结果。
  - 恢复后该工具最终**成功**，任务继续。
- 判定：看到自动重试且最终恢复 = ✅。
- 实际结果：__________ ✅/❌

### TC-D2 撞重试上限后回灌（正向-兜底）
- 步骤：让 Agent 跑一条**持续瞬时失败**的命令（**全程断网**跑 `curl https://example.com`，不恢复）。
- 预期：自动重试 **2 次**后停止（共执行 3 次），把最后一次失败**回灌给模型**（模型据此换思路或告知你），**不无限重试、不卡死**。
- 判定：恰好 3 次执行后停、失败信息进对话 = ✅。
- 实际结果：__________ ✅/❌

### TC-D3 非瞬时失败**不**重试（反向，防误重试）
- 步骤：让 Agent 改一个文件使**单元测试断言失败**（逻辑错误），并跑测试；或直接让它跑一段必然 `AssertionError` 的代码。
- 预期：测试/命令失败后**立即回灌模型**去修，**没有** `tool_retry`、**不**退避等待（逻辑错误重试无意义）。
- 判定：无重试、直接进入修复 = ✅。
- 实际结果：__________ ✅/❌

### TC-D4 总开关可关
- 步骤：`config.yaml` 设 `auto_retry: false`，重启应用，重做 TC-D1 的断网场景（不恢复）。
- 预期：工具失败**一次即回灌**，无任何重试。
- 判定：关掉后行为回到旧版（不重试）= ✅。验完**改回 `true`**。
- 实际结果：__________ ✅/❌

### TC-D5 退避参数生效（可选）
- 步骤：`retry_max_attempts: 3`、`retry_backoff_base: 1.0`，重启，重做 TC-D2（全程断网）。
- 预期：重试 3 次，间隔约 1s、2s、4s（指数退避），共执行 4 次后停。
- 判定：次数与间隔大致符合 = ✅。验完改回 `2` / `0.5`。
- 实际结果：__________ ✅/❌

## 二、块 B/C 评估摘要条（**视觉，Linux 看不了**）

机制：可评估的工具结果（测试 / 命令 / 检索）下方多一行摘要条 `.tr-eval`：绿(ok)/橙(warn)，warn 末尾带错误分类标签 `[logic]`/`[transient_io]` 等。

### TC-B1 测试全过 → 绿条
- 步骤：让 Agent 跑一个**全部通过**的测试（如让它对一个简单函数写测试并跑）。
- 预期：工具结果下方**绿色**摘要条，形如 `3/3 通过`（或 `测试全过`）。
- 判定：绿条 + 通过数正确 = ✅。
- 实际结果：__________ ✅/❌

### TC-B2 测试失败 → 橙条 + 分类标签
- 步骤：让 Agent 跑一个**有失败**的测试（部分用例 `AssertionError`）。
- 预期：**橙色**摘要条，形如 `⚠ 2/3 通过（测试未全过=blocker）[logic]`——末尾 `[logic]` 是块 C 分类。
- 判定：橙条 + 计数对 + `[logic]` 标签 = ✅。
- 实际结果：__________ ✅/❌

### TC-B3 命令退出码
- 步骤：跑 `run_powershell` 一条**成功命令**（`echo hi`）和一条**失败命令**（`cmd /c exit 1` 或访问不存在路径）。
- 预期：成功 → 绿条 `退出码 0`；失败 → 橙条 `退出码 1（退出码非零=失败）`，含错误分类标签。
- 判定：两种退出码各自对 = ✅。
- 实际结果：__________ ✅/❌

### TC-B4 检索命中数
- 步骤：让 Agent 用 `grep_search` 搜一个**存在**的词，再搜一个**不存在**的词。
- 预期：存在 → 绿条 `命中 N 条`；不存在 → 条目显示 `返回 0 条`（不报红，空结果是事实非错误）。
- 判定：命中数显示合理 = ✅。
- 实际结果：__________ ✅/❌

### TC-B5 摘要条样式不破坏布局
- 检查点：摘要条不刷屏、不撑破对话框、深色主题下配色清晰、长文本不溢出。
- 判定：观感 OK = ✅。

# ADR-0004: 多模态范围 = 图片 + 截图 + 文档（不含语音）

- 状态：已接受
- 日期：2026-06-08

## 背景
需确定首版多模态支持的输入/输出能力边界，避免范围蔓延。

## 决策
**纳入**：
- 图片输入（粘贴/拖拽图片 → 视觉模型）。
- 截图/看屏幕（热键截屏 → 模型）。
- 文件/文档（PDF/代码/文本作为上下文）。

**排除（当前版本）**：
- 语音输入/输出（ASR/TTS）。理由：编程场景价值相对低，且引入额外引擎与延迟。

## 结果
- 省去 ASR/TTS 依赖，聚焦视觉 + 文档。
- 所有非文本输入在 `multimodal/` 归一为统一消息内容块后再进内核。
- 若后续需要语音，新增 ADR 并扩展 `multimodal/`，不影响既有架构。
